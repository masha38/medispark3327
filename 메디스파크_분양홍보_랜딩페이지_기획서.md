## 1. 한 줄 전략

**“병원 이름을 앞세운 투자 기대감”보다, 학교·공원·생활·의료 예정지를 걸어서 누리는 가족의 하루를 먼저 보여주고, 1,215세대 대단지와 공간 설계라는 실체로 신뢰를 쌓아 상담 신청으로 전환한다.**

메인 콘셉트는 **모든 일상이 가까워지는 브레인시티의 중심 생활**이다. 의료 복합타운은 강한 관심 유발 요소지만 개원 전 예정 사업이므로, 확정적 가치 상승 표현 대신 “계획 기준”, “예정”, “관계기관 사정에 따라 변경 가능”을 명확히 표기한다.

  - `이름` — 필수, 이름 입력
  - `전화번호` — 필수, 휴대전화 번호 입력 및 숫자 형식 검증
  - `문의` — 필수, 문의 내용 또는 희망 방문 일정 입력
- 시스템 저장값: `개발자` — **장문희**. 고객 입력란으로 노출하지 않고 프런트엔드 상수와 Apps Script 서버 측 기본값으로 기록한다.
- 개인정보 수집·이용 동의는 별도 필수 체크 항목으로 제공하고 `보기`를 누르면 전문을 확인할 수 있게 한다.
- 제출 버튼: `방문상담 신청하기`
- 마이크로카피: `상담 신청은 청약 및 계약 의무가 아닙니다.`
- 제출 성공 문구: `방문상담 신청이 완료되었습니다. 담당자가 확인 후 연락드리겠습니다.`
- 제출 중에는 버튼을 비활성화해 중복 신청을 방지하고, 오류 발생 시 해당 입력란 바로 아래에 안내한다.
- 데스크톱에서는 중앙 모달, 모바일에서는 화면 하단 바텀시트로 열리게 하며 닫기 버튼과 바깥 영역 닫기를 지원한다.

#### Google Apps Script·구글시트 저장

- 방문상담 폼 제출 시 Google Apps Script 웹앱의 `doPost(e)` 엔드포인트로 전송한다.
- Apps Script는 요청값을 검증한 뒤 지정된 구글시트의 마지막 행에 추가한다.
- 개발자 정보는 클라이언트가 보낸 값만 신뢰하지 않고 Apps Script에서 `장문희`로 강제 지정해 변조를 방지한다.
- 권장 시트명: `방문상담_DB`
- 저장 컬럼:

| 컬럼 | 값 | 수집 방식 |
|---|---|---|
| 접수일시 | Apps Script 실행 시각 | 서버 자동 기록 |
| 이름 | 고객 입력값 | 필수 |
| 연락처 | 고객 입력값 | 필수 |
| 문의 | 고객 입력값 | 필수 |
| 개발자 | 장문희 | 서버 자동 기록 |
| 유입페이지 | 현재 페이지 URL | 자동 기록 |
| UTM Source | `utm_source` | 값이 있을 때 자동 기록 |
| UTM Campaign | `utm_campaign` | 값이 있을 때 자동 기록 |

- 전화번호는 표시용 형식으로 정규화하고, 이름·문의에는 HTML 태그 제거 및 스프레드시트 수식 주입 방지 처리를 적용한다.
- 성공 시 `{ "success": true }`, 실패 시 개인정보가 포함되지 않은 오류 코드와 메시지를 반환한다.
- 제출 버튼 중복 클릭 방지, 요청 타임아웃, 실패 시 재시도 안내를 구현한다.
- Apps Script 배포 URL과 스프레드시트 ID는 소스에 직접 노출하지 않도록 배포 환경 설정 또는 Apps Script 속성 저장소를 사용한다.
- 구글시트 접근 권한은 상담 담당자 등 필요한 계정으로 제한하고, 보유기간 경과 데이터의 삭제 기준을 개인정보처리방침에 명시한다.
