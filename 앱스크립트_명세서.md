# 메디스파크 방문상담 Apps Script 명세서

## 1. 목적

랜딩페이지 방문상담 폼에서 고객의 이름, 연락처, 문의를 수신해 구글시트 `방문상담_DB`에 저장한다. 개발자 정보는 브라우저 값과 무관하게 서버에서 `장문희`로 강제 기록한다.

## 2. 구성

- 클라이언트: `index.html`, `script.js`, `config.js`
- 서버: Google Apps Script 웹 앱 `Code.gs`
- 저장소: 구글 스프레드시트의 `방문상담_DB` 시트
- 요청 형식: HTTPS POST, JSON 본문, `Content-Type: text/plain;charset=utf-8`
- 응답 형식: JSON

## 3. POST 요청

### 엔드포인트

Apps Script 웹 앱 배포 후 발급되는 `/exec` URL

### 요청 필드

| 필드 | 형식 | 필수 | 최대 길이 | 설명 |
|---|---|---:|---:|---|
| `name` | string | 예 | 40 | 고객 이름 |
| `phone` | string | 예 | 13 | `010-0000-0000` 형식 |
| `inquiry` | string | 예 | 1,000 | 문의 또는 방문 희망 내용 |
| `developer` | string | 아니오 | - | 전송되지만 서버는 무시하고 `장문희` 기록 |
| `pageUrl` | string | 아니오 | 500 | 신청 페이지 URL |
| `utmSource` | string | 아니오 | 100 | 광고 유입 소스 |
| `utmMedium` | string | 아니오 | 100 | 광고 매체 유형 |
| `utmCampaign` | string | 아니오 | 100 | 캠페인명 |
| `userAgent` | string | 아니오 | 500 | 브라우저 정보 |

### 요청 예시

```json
{
  "name": "홍길동",
  "phone": "010-1234-5678",
  "inquiry": "84A 타입 방문상담을 희망합니다.",
  "developer": "장문희",
  "pageUrl": "https://example.com/?utm_source=naver",
  "utmSource": "naver",
  "utmMedium": "cpc",
  "utmCampaign": "braincity",
  "userAgent": "Mozilla/5.0"
}
```

## 4. 응답

성공:

```json
{"success":true,"error":null,"receiptId":"UUID"}
```

실패:

```json
{"success":false,"error":"INVALID_INPUT"}
```

| 오류 코드 | 의미 |
|---|---|
| `INVALID_INPUT` | 필수값 누락 또는 전화번호 형식 오류 |
| `SERVER_ERROR` | 시트 설정, 권한 또는 서버 실행 오류 |

## 5. 저장 컬럼

`접수일시 → 이름 → 연락처 → 문의 → 개발자 → 상담상태 → 유입페이지 → UTM Source → UTM Medium → UTM Campaign → 접수ID → User Agent` 순서로 저장한다.

- `접수일시`: Apps Script 서버 시각
- `개발자`: 항상 `장문희`
- `상담상태`: 신규 접수 시 `신규`
- `접수ID`: Apps Script에서 생성한 UUID

## 6. 검증 및 보안

- 이름과 문의에서 HTML 태그를 제거한다.
- `=`, `+`, `-`, `@`로 시작하는 값은 앞에 작은따옴표를 붙여 시트 수식 주입을 방지한다.
- 전화번호는 한국 휴대전화 형식을 검사한다.
- Script Lock으로 동시 저장 시 행 충돌을 방지한다.
- 스프레드시트 ID는 코드가 아닌 Script Properties에 저장한다.
- 구글시트 공유 권한은 실제 상담 담당 계정으로 제한한다.
- Apps Script 웹 앱 URL은 공개 엔드포인트이므로 스팸 방지를 위해 운영 단계에서 CAPTCHA, 허니팟 또는 호출량 제한을 추가하는 것을 권장한다.

## 7. 배포 절차

1. 구글시트를 생성하고 URL에서 스프레드시트 ID를 확인한다.
2. Apps Script 프로젝트에 `Code.gs`를 붙여넣는다.
3. 프로젝트 설정 → 스크립트 속성에 `SPREADSHEET_ID`를 등록한다.
4. `setupSheet()`를 한 번 실행하고 권한을 승인한다.
5. 배포 → 새 배포 → 유형 `웹 앱`을 선택한다.
6. 실행 사용자 `나`, 액세스 사용자는 랜딩페이지 운영 조건에 맞게 설정한다.
7. 발급된 `/exec` URL을 `config.js`의 `APPS_SCRIPT_URL`에 입력한다.
8. 테스트 신청 후 구글시트 저장값과 접수 ID를 확인한다.

코드를 수정하면 기존 배포를 새 버전으로 업데이트해야 한다. 테스트용 `/dev` URL이 아니라 운영용 `/exec` URL을 사용한다.

## 8. 운영 점검

- Apps Script 실행 기록에서 오류율 확인
- 상담상태를 `신규 → 연락완료 → 방문예약 → 방문완료 → 종료`로 관리
- 개인정보 보유기간에 따라 정기 파기
- 폼 및 시트 접근 권한 변경 이력 관리
